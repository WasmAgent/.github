# WasmAgent 2026 Q3 Development Boundaries

Primary outcome:
Independent repository progress with a reproducible
Protect → Record → Audit → Admit integration path.

Development principles:
- every capability has one primary owner
- shared schemas have one canonical source
- repositories integrate through versioned contracts
- adapters are preferred over copied implementations
- new repositories and packages require explicit ownership
- research projects may evolve independently
- no central process may block normal development indefinitely

Architecture review required for:
- a new public schema or protocol
- duplicated cross-repository capability
- a breaking contract change
- a new repository
- a new package with cross-repository responsibility
